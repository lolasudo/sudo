# Практическое занятие №11-12
## Проработка ценностного предложения и функциональности своего решения

**Тема проекта:** Zero.id — AI-Native платформа визуального конструирования атак

---

## Шаг 1. Актуализация ценностного предложения

### Операционализированное ценностное предложение

> **Для руководителей отделов пентестинга в крупных компаниях (CISO)** в ситуации **ежеквартальной проверки защищённости перед регулятором или после новостей о новых атаках** решение обеспечивает **полную картину уязвимостей за 2-3 часа вместо 2-4 недель**, улучшая **скорость получения результатов в 20 раз** по сравнению с **наймом внешних пентестеров**, при условиях:
> - Интеграция с существующей инфраструктурой (AD, почтовые серверы)
> - Режим read-only (без риска для production)
> - Соответствие требованиям импортозамещения

### Почему пользователь поверит (основания)

| Основание | Пояснение |
| :--- | :--- |
| **1. SafeBreach подтверждает тренд** | Мировой лидер BAS объявил о переходе к AI-First (февраль 2026) |
| **2. Рыночный запрос** | 76% атак уже используют ИИ — нужна асимметричная защита |
| **3. Дефицит кадров** | Пентестеров не хватает, они дороги (от 300 тыс/мес) |
| **4. Импортозамещение** | Западные вендоры ушли — нужны российские аналоги |

## Шаг 2. Пользовательские сценарии (User Flow)

### Основной сценарий: «Первая проверка защищённости»

| Шаг | Действие | Результат |
| :--- | :--- | :--- |
| **1** | CISO получает требование от регулятора провести оценку защищённости | Триггер |
| **2** | Заходит в Zero.id, создаёт новый проект | Вход в систему |
| **3** | Подключает интеграцию с Active Directory и почтовым сервером | Система получает данные об инфраструктуре |
| **4** | Открывает визуальный конструктор | Интерфейс для сборки |
| **5** | Перетаскивает блоки: Фишинг + Дипфейк + Эксплойт | Собирает сценарий |
| **6** | Запускает симуляцию в режиме read-only | Начинается тестирование |
| **7** | Система выполняет атаки, собирает результаты | Идёт процесс |
| **8** | Получает уведомление о завершении | Оповещение |
| **9** | Открывает отчёт с картой уязвимостей и рекомендациями | Получение результата |
| **10** | Формирует PDF-отчёт для регулятора | Фиксация результата |

### Альтернативный сценарий: «Повторное тестирование после исправлений»

| Шаг | Действие | Результат |
| :--- | :--- | :--- |
| **1** | CISO получает уведомление, что патчи установлены | Триггер |
| **2** | Открывает ранее созданный проект | Вход |
| **3** | Выбирает «Запустить повторно» | Быстрый запуск |
| **4** | Система запускает тот же сценарий | Повтор теста |
| **5** | Получает новый отчёт | Результат |
| **6** | Сравнивает с предыдущим отчётом (график «было/стало») | Анализ динамики |
| **7** | Отправляет отчёт руководству | Фиксация |

### Альтернативный сценарий: «Ошибка интеграции»

| Шаг | Действие | Результат |
| :--- | :--- | :--- |
| **1** | CISO подключает интеграцию с AD | Действие |
| **2** | Система не может получить данные (неверные права) | Ошибка |
| **3** | Выводит понятное сообщение: «Нет прав доступа к AD» | Обратная связь |
| **4** | Предлагает инструкцию по настройке | Помощь |
| **5** | CISO исправляет настройки | Решение проблемы |
| **6** | Повторяет попытку | Успех |

## Шаг 4. Backlog MVP (User Stories + Acceptance Criteria)

| ID | User Story | Acceptance Criteria | Priority |
| :--- | :--- | :--- | :--- |
| **US1** | Как CISO, я хочу создать новый проект, чтобы начать тестирование | — Поле «Название проекта» обязательно<br>- Кнопка «Создать» активна<br>- После создания открывается проект | Must |
| **US2** | Как пентестер, я хочу подключить Active Directory, чтобы система автоматически получила данные о пользователях и устройствах | — Проверка доступности AD<br>- Понятное сообщение об ошибке<br>- Индикатор успешного подключения | Must |
| **US3** | Как пентестер, я хочу перетаскивать блоки атак в конструктор, чтобы собрать сценарий | — Блоки перетаскиваются<br>- Соединяются стрелками<br>- Можно удалить блок | Must |
| **US4** | Как пентестер, я хочу настроить параметры фишинга (тема письма, отправитель), чтобы сделать атаку реалистичной | — Поля доступны для редактирования<br>- Можно выбрать получателей из AD | Must |
| **US5** | Как CISO, я хочу запустить симуляцию в режиме read-only, чтобы не навредить production | — Кнопка «Запуск» активна<br>- Подтверждение режима read-only<br>- Статус «Выполняется» | Must |
| **US6** | Как CISO, я хочу видеть прогресс выполнения в реальном времени | — Индикатор выполнения<br>- Лог действий | Should |
| **US7** | Как CISO, я хочу получить отчёт с картой уязвимостей после завершения | — Отчёт открывается автоматически<br>- Карта с путями атак<br>- Список уязвимостей | Must |
| **US8** | Как CISO, я хочу экспортировать отчёт в PDF, чтобы отправить регулятору | — Кнопка «Экспорт PDF»<br>- Файл скачивается<br>- Открывается корректно | Must |
| **US9** | Как пентестер, я хочу сохранить сценарий, чтобы использовать его повторно | — Кнопка «Сохранить»<br>- Сценарий появляется в списке | Should |
| **US10** | Как CISO, я хочу видеть историю запусков, чтобы отслеживать динамику | — Список предыдущих запусков<br>- Дата, статус, ссылка на отчёт | Could |

## Шаг 5. Нефункциональные требования (6-10)

| ID | Категория | Требование | Критичность |
| :--- | :--- | :--- | :--- |
| **NFR1** | Безопасность | Все симуляции выполняются в режиме **read-only** — никаких изменений в production |  Критично |
| **NFR2** | Безопасность | Доступ к платформе только по 2FA |  Критично |
| **NFR3** | Производительность | Запуск сценария — не более 5 секунд после нажатия | Высокая |
| **NFR4** | Производительность | Отчёт формируется не более 10 секунд после завершения | Высокая |
| **NFR5** | Интеграция | Поддержка русскоязычных версий: AD, MaxPatrol, Astra Linux | Высокая |
| **NFR6** | Надёжность | Доступность платформы 99.5% (кроме плановых работ) | Средняя |
| **NFR7** | Масштабируемость | Поддержка одновременной работы 50+ проектов | Средняя |
| **NFR8** | Локализация | Интерфейс на русском языке (технические термины — англ) | Средняя |
| **NFR9** | Аудит | Логирование всех действий пользователя | Высокая |
| **NFR10** | Совместимость | Поддержка российских ОС (Astra Linux, Alt Linux) |  Критично |
